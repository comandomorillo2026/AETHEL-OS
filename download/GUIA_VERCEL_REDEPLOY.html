🚀 Cómo Re-desplegar en Vercel

Guía paso a paso para arreglar deployments fallidos de NexusOS

⚠️ Si el deployment falla automáticamente:

Los deployments automáticos desde GitHub pueden fallar por varias razones. Aquí te explico cómo forzar un nuevo deployment manual.

1 Ir a Vercel Dashboard

Abre tu navegador y ve a:

https://vercel.com/comandomorillo2026/projects

O directamente a tu proyecto:

https://vercel.com/comandomorillo2026/nexus-os
2 Seleccionar el Proyecto

Tus proyectos:

📁 nexus-os ❌ Error
📁 otro-proyecto ✓ Activo

Haz click en "nexus-os" para entrar al proyecto.

3 Ver los Deployments

4 Forzar Redeploy

En la página de Deployments, busca el deployment fallido y haz click en los tres puntos "...":

🔧 Método Alternativo: Desde Settings

A Ir a Settings → Git
General
Git
Environment Variables

En Settings → Git puedes ver la conexión con GitHub.

B Desconectar y Reconectar

Si el problema persiste, puedes:

  1. Ir a Settings → Git
  2. Hacer click en "Disconnect"
  3. Reconectar con GitHub
  4. Seleccionar el repositorio NexusOS

🔐 Variables de Entorno Necesarias

! Verificar Variables en Vercel

Ve a Settings → Environment Variables y asegúrate de tener:

NEXTAUTH_SECRET = (cualquier string largo aleatorio)
NEXTAUTH_URL = https://nexus-os-alpha.vercel.app
DATABASE_URL = (tu conexión PostgreSQL)
GITHUB_TOKEN = (opcional, para métricas)
⚠️ Importante: Si falta DATABASE_URL, el deployment fallará.

📊 Ver Logs del Error

? Ver qué causó el error

Para ver exactamente qué falló:

  1. Ve a la pestaña "Deployments"
  2. Haz click en el deployment que dice "Failed"
  3. Verás los logs del build
  4. Busca líneas rojas con "Error" o "Failed"

Build Logs:

✓ Compiled successfully
✓ Generating static pages
Error: Something went wrong...
Build failed with exit code 1
✓ ¿Necesitas ayuda?

Copia el mensaje de error de los logs y envíamelo para ayudarte a identificar el problema exacto.
